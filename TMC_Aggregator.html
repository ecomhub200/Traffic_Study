How it works
Pick your corridor folder — the tool walks every subfolder, keeps only .csv files, groups multiple count days per intersection, computes averaged peak-hour movements, and verifies parser accuracy against published values.

About the browser dialog: when you pick a folder, the browser will list every file inside (videos, images, etc.). Only .csv files are actually loaded — everything else is ignored. Nothing leaves your computer.

How it works
Drop crash CSV files exported from CrashLens. The tool reads severity, collision type, weather, light condition, location, and temporal fields, groups crashes by Node ID, and builds corridor + per-intersection crash summaries. If a non-standard CSV is detected, you'll be asked to map the columns — same as the TMC workflow.

Combined reporting: when both TMC and crash data are loaded for the same corridor, a Combined Report becomes available — pairing peak-hour volumes with crash history at each intersection.
